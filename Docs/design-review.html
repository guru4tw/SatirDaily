設計健檢 Design Review

SatirDaily 改進建議
八位專家的視角

針對 guru4tw.github.io/SatirDaily 的設計與體驗,從五位薩提爾取向專家與三位使用者體驗／設計大師的框架出發,提出可落地的改進方向。

檢視日期 2026-06-23　·　涵蓋:內容定位、情感溫度、易用性、無障礙、SEO 與技術體驗

關於這份文件的方法: 以下「專家視角」是以每位人物公開著作與既有理論框架為分析透鏡(例如冰山理論、Nielsen 十大可用性原則、Rams 好設計十誡),套用到本網站上做評析,並非引用其本人對本站發表的真實言論。目的是用多元、互補的專業角度,幫你看見不同層面的改進空間。

薩提爾取向 · 內容與情感溫度

這五個視角關注的是:訪客「為什麼」會來找薩提爾活動,以及網站能否在第一眼就接住那份渴望。呼應你 CLAUDE.md 的宗旨——「宗旨、內容、正面回饋為呈現重點,不要弱化成冷冰冰條列」。

維琴尼亞·薩提爾 Virginia Satir薩提爾模式創始人

透鏡:自我價值、一致性、冰山、五種自由——「問題本身不是問題,如何應對才是問題。」每個人都有價值、都能經由自我認可界定自己。

看見什麼

  • 站名就叫 SatirDaily,卻沒讓訪客感受到薩提爾的核心精神——每個人都有價值、都值得被看見。網站把活動當資料陳列,少了「來這裡是為了更靠近自己」的定調。
  • 訪客搜「薩提爾活動」,水面下的渴望往往是連結、成長、被理解;目前首頁主標僅「薩提爾」、副標「台灣薩提爾活動資訊!」,停在最表層的行為(找活動),沒接住底層的渴望。

建議

  • 用一句帶薩提爾精神的話定調首頁,讓站有靈魂而非冷清單,例如呼應「五種自由」或「一致性」:在台灣,找一場讓自己更靠近自己的薩提爾活動。
  • 把一致性原則套到文案:誠實、溫暖、不浮誇——標語別過度承諾,如實呈現資料現況,內外一致。
溫暖代表金句 · 三選一
  1. 「我能得到的最大禮物,是被你看見、被你聽見、被你理解、被你觸碰;而我能給的最大禮物,也是如此地回應另一個人。」原話　·　獨特:看見與連結
  2. 「我就是我。在全世界,沒有一個人和我完全一樣……我屬於我自己,所以我能塑造我自己。我就是我,而我很好。」原話,〈自我價值宣言〉　·　獨特:自我價值
  3. 「問題本身不是問題,如何面對問題,才是問題。」原話　·　獨特:面對的姿態
瑪莉亞·葛茉莉 Maria Gomori體驗式家族治療 · 薩提爾嫡傳

透鏡:人的臨在、自發與創造——與薩提爾共事二十年,重視真實的人與真實的接觸,而非技術或標籤;「成為更完整的人」。

看見什麼

  • 活動由「帶領者」這個人撐起,但目前帶領者只是 meta 列裡的一個小標籤,沒有面孔、沒有溫度——人被資料化了。
  • 整體視覺已經很溫柔(湖水青、襯線字、水面意象),方向正確;但文字語氣偏「公告」,少了一點對訪客的招呼與臨在感。

建議

  • 強化帶領者的存在感:可加入帶領者一句話介紹或頭像(若版權允許),或讓「帶領者」變成可篩選/可點的標籤,聚合同一位老師的所有活動。
  • 在空狀態與載入狀態用溫暖的人話,而非系統訊息。例如找不到活動時,給一句「這個分類暫時沒有活動,換個關鍵字,或看看『日期未定』的常態課程」。
溫暖代表金句 · 三選一
  1. 「你不是沒有選擇,而是你『認為』自己沒有選擇。」原話　·　獨特:選擇/自由
  2. 「每個人,都有機會重新成為自己。」原話　·　獨特:重新開始的盼望
  3. 「改變,任何時候開始都不嫌晚。」理念整理(呼應她百歲仍工作、《愛與自由》的精神)　·　獨特:終生成長
約翰·貝曼 John Banmen薩提爾系統轉化治療

透鏡:冰山與系統轉化——行為底下是感受、觀點、期待與渴望;治療(與溝通)的目標是「從問題走向可能與希望」。

看見什麼

  • 活動卡的 highlights(正評)設計很好,正是「希望感」的具體呈現;這是全站最有溫度的元素,值得更突出。
  • 站上資訊大多停在「有什麼活動」(行為層),少了「參加後可能帶來什麼轉變」(渴望/可能層)的訊息,難讓人從觀望走向行動。

建議

  • 讓 正評優先曝光:在近期活動最上方放一張「本週推薦」,或讓有正評的卡片加微標記,把希望感放到水面上。
  • 把文案從「資訊」升級為「可能」:卡片或首頁帶一句指向轉變的話,例如「找一段讓自己更靠近自己的旅程」。
溫暖代表金句 · 三選一
  1. 「我們要先看見自己內心的美好,才可能真正看見別人;先感受到對自己的愛,才能去愛別人。」原話　·　獨特:先愛自己
  2. 「我常常回到我的冰山。」原話　·　獨特:向內覺察的修練
  3. 「成長的方向,是從問題走向可能與希望。」理念整理(呼應其「系統轉化」取向)　·　獨特:轉化/希望
成蒂薩提爾嫡傳 ·體驗式家族,婚姻伴侶治療

透鏡:關係中的連結與自我價值——師承薩提爾、葛茉莉、貝曼近三十年,曾任葛茉莉助手;著《我們之間》。人在關係裡受傷,也在關係裡療癒。

體驗式家族治療 · 薩提爾嫡傳:承葛茉莉一脈的體驗式取向——治療不只在「談」問題,更在現場「經驗」轉變(雕塑、角色、當下接觸)。成蒂是少數華人國際認證的薩提爾模式訓練師,把這套以人為本、在關係中流動的工作方式帶進台灣,強調治療師自身的一致與臨在。

看見什麼

  • 很多訪客找活動是為了關係議題(親子、伴侶、家庭),但站上活動只有單一「地區」維度,無法用「我想處理的關係」來找。
  • highlights(正評)呈現了個人的轉變,但少了「關係因此有什麼不同」的角度——而那常是訪客真正在意的。

建議

  • 加「議題/關係」分類:伴侶、親子、家庭、自我成長,讓訪客從「我的關係困境」入手,而非從機構名稱猜。
  • 正評可兼採關係層面的改變(不只「我變好了」,也有「我們之間變好了」),呼應薩提爾在關係中流動的愛。
溫暖代表金句 · 三選一
  1. 「親密關係,是人生最好的機會,去遇見自己與對方真實的自我。」出自《我們之間》　·　獨特:關係作為相遇
  2. 「親密,是能分享脆弱——那最容易受傷的地方,也是最能療傷的地方。」出自《我們之間》　·　獨特:脆弱與療癒
  3. 「從『我和你』,慢慢走向『我們』。」出自《我們之間》　·　獨特:成為「我們」
李崇建薩提爾對話 · 台灣推廣

透鏡:對話的普及化——把薩提爾從治療室帶進親子、師生、日常,用一般人聽得懂的語言降低門檻。

看見什麼

  • 很多訪客是初次接觸的家長或老師,可能不確定「薩提爾」是什麼、適不適合自己。站上沒有任何給新手的入口或一句說明。
  • 站名 SatirDaily 與「每日匯整」很清楚,但受眾(這是給誰的?)沒有明說。

建議

  • 加一段極短的「什麼是薩提爾?適合誰?」(2–3 句、可收合),用日常語言:這是一種關於溝通與自我成長的取向,常見於親子、伴侶、教學與自我探索。
  • 用情境式的關鍵字快搜取代純文字框,例如預設標籤:親子溝通、自我成長、伴侶、教師研習、線上課,讓新手不必想關鍵字就能開始。
溫暖代表金句 · 三選一
  1. 「好奇,是傾聽的起點、改變的緣起、接納的開始;好奇最終之處,是連結人的生命力。」原話　·　獨特:好奇
  2. 「真正的溝通,不是解決問題,而是與人連結。」理念整理(其著作核心精神)　·　獨特:連結勝於解決
  3. 「對話時,先不急著給答案、講道理、解決問題——先連結那個人。」理念整理(其對話方法)　·　獨特:對話姿態

使用者體驗 · 易用性與設計品質

這三個視角關注的是:訪客能不能順、快、放心地找到並信任一場活動,以及網站在技術與細節上是否扎實。

唐·諾曼 Don Norman以使用者為中心的設計

透鏡:可發現性、回饋、心智模型——好的設計讓人一看就知道能做什麼、做了之後有清楚回應。

看見什麼

  • 讀取 events.json 時沒有載入回饋(無骨架/轉圈),慢網路下會看到一片空白;失敗時才出現訊息。系統狀態不可見。
  • 篩選只有「地區」一個下拉,但排序選單裡卻有帶領者/主辦——能排序卻不能篩選,心智模型不一致。
  • 分頁(近期/日期未定/已過去)切換會悄悄改排序,使用者不一定察覺到發生了什麼。

建議

  • 加入載入骨架(skeleton)或轉圈+ 友善錯誤狀態,讓每個動作都有即時回饋。
  • 把「帶領者、主辦」也做成篩選(下拉或標籤),讓排序與篩選的能力對齊。
  • 切換分頁時,可加一行微動畫或提示,讓排序改變被「看見」。
雅各布·尼爾森 Jakob Nielsen可用性十大原則 (NN/g)

透鏡:十大可用性啟發法——系統狀態可見、貼近真實世界、辨識優於回憶、容錯、美觀且精簡、無障礙。

看見什麼

  • 對比度:次要文字(資料更新日、來源)用的淺灰綠(原 #8b9b96)在白底上小字未達 WCAG AA,視力較弱者吃力。
  • 鍵盤無障礙:分頁用了 role="tab" 但沒有 aria-controls / tabpanel,也沒有左右方向鍵切換,螢幕報讀與鍵盤使用者體驗不完整。
  • 辨識優於回憶:搜尋是自由文字,使用者得自己想關鍵字;沒有常用標籤可點。
  • 重新整理後,使用者的篩選/搜尋條件不會被記住(無狀態保存)。

建議

  • 把次要文字色加深至可過 AA(約 #5d6f6b 以上),確保正文與標籤皆達 4.5:1。
  • 補齊 tab 無障礙:aria-controls 連到 tabpanel、支援方向鍵、focus 樣式清楚。
  • 用 localStorage 記住使用者上次的分頁、地區與排序,回訪更順。
迪特·拉姆斯 Dieter Rams好設計十誡

透鏡:誠實、極簡、注重細節、歷久——好設計「盡可能少設計」,且內容才是主角。

看見什麼

  • 誠實原則:首頁標榜「每天自動更新」「每日匯整」,但目前只有約 77 筆且部分常態課程無日期;若某天爬蟲沒新資料,標語仍說「每日」,可能與實況不符。
  • 視覺語言乾淨克制,水面 logo 巧妙——這部分已做得很好,維持即可。
  • 細節:行動版隱藏 hero 圖、sticky 控制列等都處理得不錯;但卡片在「已過去」用整體變淡(opacity .62)會連帶降低文字對比。

建議

  • 讓標語誠實對應實況:顯示「最後更新 X 天前」與資料總筆數(已有更新日,可再加總數與健康狀態),別讓「每日」變成空話。
  • 「已過去」卡片改用更克制的降權(例如僅左側色條轉灰 + 加「已結束」標,而非整體調淡),保住可讀性。
  • 持守「內容是主角」:抗拒加入裝飾性元素,把心力放在資料的完整與正確。

綜整 · 建議優先順序

把六個視角收斂成一份可執行清單。快速見效 多為前端小改;較大投入 需流程或架構調整。

1改寫首頁文案,接住訪客的渴望主標+副標傳達「找一段成長/連結」,而非僅「活動資訊」。(貝曼 · 李崇建)快速見效
2加新手入口「什麼是薩提爾?適合誰?」2–3 句可收合說明 + 情境快搜標籤,降低初接觸門檻。(李崇建)快速見效
3提高文字對比、補齊鍵盤/讀屏無障礙加深次要文字色至 WCAG AA;tab 加 aria-controls 與方向鍵。(尼爾森)快速見效
4載入骨架 + 友善空/錯誤狀態讓系統狀態隨時可見,訊息用人話。(諾曼 · 葛茉莉)快速見效
5讓標語誠實對應資料實況顯示「最後更新 X 天前 + 總筆數」;「已過去」卡片改克制降權。(拉姆斯)快速見效
6帶領者、主辦也做成篩選/可點標籤排序與篩選能力對齊;聚合同一位老師的活動。(諾曼 · 葛茉莉)快速見效
7記住使用者的分頁/地區/排序用 localStorage 保存上次條件,回訪更順。(尼爾森)快速見效
8每場活動加「加入行事曆 (.ics)」活動網站的高價值功能;可再考慮整站訂閱(LINE/Email/行事曆 feed)。較大投入
9SEO 與社群分享:預先渲染 + 結構化資料目前活動由 JS 載入,爬蟲看不到內容;加 Event JSON-LD、OG/Twitter 卡,或在 build 時把活動寫進 HTML。較大投入
10突出「正評/本週推薦」與帶領者面孔把最有溫度的元素放到水面上,呼應宗旨不淪為冷條列。(貝曼 · 葛茉莉)較大投入

已經做得好的地方

改進建議之外,也記下值得保留的優點,別在重構時弄丟。

  • 視覺語言溫柔且一致:湖水青配色、襯線標題、水面/倒影 logo,氣質貼合主題。
  • 連結驗證徽章(✓ 連結已驗證)與「來源連回原始頁」建立了難得的信任感。
  • 分頁切換自動套用合適排序、每分頁有說明 hint,是體貼的小細節。
  • 多關鍵字搜尋、地區篩選、響應式版面,核心功能完整可用。
