SatirDaily SEO 深度優化建議報告

針對全靜態 GitHub Pages 架構的搜尋引擎優化策略彙整

1. 核心現況診斷

SatirDaily 目前已完成基礎的技術型 SEO（如預渲染、JSON-LD、Sitemap），在 GitHub Pages 的限制下表現優異。然而，目前的「單頁 SPA」架構是限制長尾流量增長的最大瓶頸。

指標現況描述SEO 影響
收錄頁面僅有 1 個首頁 URL無法針對特定講師、主題或地區進行排名。
內容深度77 場活動預渲染Google 可抓取內容，但缺乏主題權威性（Authority）。
結構化資料ItemList (72 筆)雖然符合規範，但單頁承載過多 Event 易被 Google 視為目錄頁而非活動頁。

2. 高價值優化清單 (Manus AI 建議)

以下建議遵循「全免費、純靜態、自動化」的原則，旨在提升長尾關鍵字覆蓋率。

A. 靜態生成獨立活動/講師/地區頁面

原理：將原本僅存在於 events.json 的資料，透過 Python 腳本在 build 時生成為獨立的 HTML 檔案。

具體實作建議：

  • 資料夾結構： /events/{id}.html, /teachers/{name}.html, /locations/{city}.html
  • URL 規劃： 使用語意化 URL，例如 /teachers/成蒂。
  • Sitemap： 將所有生成的 HTML 加入 sitemap.xml。

優先級： 最高 | 預期效益： 顯著增加長尾關鍵字（如「成蒂 薩提爾課程」）的排名機會。

B. 自動化生成社交分享圖 (og:image)

原理： 使用 Python (Pillow 庫) 在 build 階段，根據活動標題、日期、帶領者自動繪製一張活動卡片圖。

具體實作建議：

  • 在 build_site_json.py 中加入圖像生成邏輯。
  • 生成的圖片存放在 /assets/og-images/{id}.png。
  • 在對應的 HTML 中設置 <meta property="og:image">。

優先級： 中高 | 預期效益： 提升在 FB/LINE 分享時的點擊率 (CTR)。

C. 強化主題內容與內部連結策略

原理： 增加關於薩提爾模式的深度文章，並在文章中自然地連結到相關活動頁面，建立站內權威。

具體實作建議：

  • 建立一個 /blog/ 或 /guide/ 區塊，放置「如何選擇薩提爾課程」等文章。
  • 在首頁底部增加「熱門講師」或「熱門地區」連結，導向新生成的靜態頁。

優先級： 中 | 預期效益： 降低跳出率，提升使用者停留時間與搜尋引擎權重。

4. 總結

SatirDaily 已具備良好的 SEO 基礎。未來的核心應放在「從單頁到多頁」的轉型，利用 GitHub Actions 的強大自動化能力，在不增加維護負擔的前提下，將每一場活動、每一位講師都轉化為一個可被 Google 索引的入口。這將使 SatirDaily 從一個單純的工具頁，進化為一個薩提爾模式的內容樞紐。
