SatirDaily — SEO 進階改善提案

針對您所提供的全靜態 GitHub Pages 網站架構限制，以下提出在「零成本、無後端」前提下，投資報酬率（ROI）最高的 SEO 策略方案。

核心答覆：絕對值得為每場活動／講師／地區「靜態生成獨立 HTML 頁」 (SSG)。

單頁 SPA 的 SEO 上限非常明顯。將自動化資料流延伸，在 Build 階段利用 Python 讀取 JSON 並產出獨立的 HTML 檔案，是突破長尾關鍵字覆蓋率的最有效做法。這能徹底解決「單一 URL 無法乘載多元意圖」以及「ItemList JSON-LD 權重不如 Event 專屬頁」的痛點。

1. 靜態生成獨立頁面 (SSG) 策略規劃

此步驟的邏輯是在您現有的 build_site_json.py 中，加入 HTML 模板渲染功能（可使用 Python 內建的字串格式化，或引入 jinja2），將 events.json 中的每一筆資料輸出為實體檔案。

A. 資料夾結構與 URL 規劃

  • 活動專屬頁： /events/{date}-{organizer}-{id}.html
    (例如: /events/2026-06-25-lopwilldo-workshop.html)
  • 講師/機構彙整頁： /organizers/{organizer-name}.html

B. 內部連結與 SPA 共存策略

在首頁的 SPA 列表中，卡片的點擊網址（<a href="...">）必須寫死指向靜態生成的 HTML 頁面。您可以保留 SPA 的流暢體驗：利用 JavaScript 攔截點擊事件 (e.preventDefault())，在畫面上彈出 Modal 顯示活動詳情。這樣一來，爬蟲爬到的是真實的長尾 URL，而使用者享受的是無縫的 SPA 體驗。

C. SEO 元件配置

  • 獨立 Title / Meta： 「2026 薩提爾工作坊 - {活動名稱}｜{機構名稱}」
  • 獨立 JSON-LD： 捨棄首頁龐大的 ItemList，在各獨立頁注入單一且完整的 Event Schema.org 結構化資料，極大化觸發 Google Event Rich Result 的機率。
  • Sitemap 更新： Python 腳本在產生 HTML 的同時，將這些新 URL 寫入 sitemap.xml。

3. 關於反向連結 (Backlinks) 的冷啟動

新站零反向連結是硬傷。既然 SatirDaily 是自動幫各機構推廣活動的公益平台，您可以主動出擊：

  1. 整理一份精美的 Email 或 FB 訊息。
  2. 聯繫您所爬取的 5 大來源機構（lopwilldo, satir_org 等）。
  3. 告知他們：「我們建立了一個公益的薩提爾活動匯總站，每天自動幫貴單位的活動增加曝光。如果方便，是否能在貴單位的『相關資源』或『友站連結』中提及 SatirDaily？」

只要有 1~2 個高權重的相關機構給予 dofollow 連結，對新站的排名將是質的飛躍。

維護成本與風險提示：
改用靜態生成 (SSG) 後，GitHub 儲存庫的檔案數量會隨著活動增加而變多。建議在 .gitignore 的策略上稍作調整，或者將生成的 /events/ HTML 檔案僅存在於 gh-pages 分支，保持 main 分支乾淨。
