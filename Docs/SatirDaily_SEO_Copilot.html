摘要

SatirDaily 是「全台薩提爾活動／課程每日自動匯整清單」，目前 SEO 基礎已完成，瓶頸在於 站外權重不足 與 單頁 SPA 長尾覆蓋上限。

已知弱點

  • 單頁 SPA，缺乏獨立落地頁，長尾關鍵字覆蓋有限。
  • 使用 GitHub Pages 子網域，品牌信任度不足。
  • 新站，零反向連結，外部引用缺乏。
  • 無 og:image，社群分享 CTR 受限。
  • 內容深度偏薄，主題權威性不足。
  • 過期活動混入，降低「即將舉行」相關性。
  • JSON-LD 為 ItemList，缺乏單一 Event rich result。

SEO 改善需求清單

項目效益靜態實作方式風險/成本優先級
提交 sitemap.xml 至 GSC 加速收錄，提升索引完整度 人工登入 GSC 提交 低 高
建立每活動/講師/地區靜態頁 擴張長尾關鍵字，提升 Event rich result 機率 Python build 腳本生成 /events/{id}.html，並更新 sitemap 與內部連結 維護成本中等，需處理過期活動 中高
加 og:image 提升社群分享 CTR，增加圖片搜尋曝光 build 腳本自動產活動卡圖，寫入 meta 需設計模板，成本中 中
擴充主題知識內容 提升主題權威性，抗 thin content 新增「薩提爾模式介紹」「講師專頁」靜態文檔 需人工撰寫，持續更新 中
爭取反向連結 提升網域權重，改善排名 聯繫活動主辦單位，請求引用 人工溝通，成本高 高

建議的資料夾結構

/index.html
/events/{event-id}.html
/lecturers/{lecturer-name}.html
/regions/{region-name}.html
/sitemap.xml

內部連結策略：首頁活動卡片 → 對應活動頁；講師名 → 講師頁；地區 → 地區頁。確保 sitemap.xml 每日更新。
