摘要（Grok TL;DR）

您的網站基礎 SEO 已非常扎實，尤其是預渲染與結構化資料解決了 SPA 最大痛點。目前瓶頸在長尾關鍵字、站外權重與內容深度。最值得立即行動：提交 sitemap、生成 og:image、過濾過期活動、擴充靜態內容。

靜態多頁生成：值得為講師與地區生成獨立頁（中高優先），但先從簡單版本開始。以下是詳細建議。

1Grok 高優先改善清單

1. 提交 sitemap.xml 到 GSC + 網址審查

(a) 效益：立即提升收錄率，讓 Google 發現所有預渲染活動。

(b) 實作：GSC 介面提交。

(c) 成本：低。

(d) 優先級 最高

2. 生成 og:image 與活動卡圖

(a) 效益：提升社群 CTR 與分享率；圖片搜尋曝光。

(b) 實作：Python build 腳本中使用 Pillow 或簡單 Canvas 生成帶活動資訊的 PNG，存到 /images/ 資料夾，build 時動態更新 meta og:image。

(c) 成本：中（需加依賴）。

(d) 優先級 高

3. 過濾過期活動 + 過去活動歸檔頁

(a) 效益：提升相關性訊號，避免稀釋即將舉行活動。

(b) 實作：build 腳本分離 upcoming / past events，生成 /past.html 或 /archive/ 目錄。

(c) 成本：低。

(d) 優先級 高

2靜態多頁生成建議

值得做：是的，尤其是講師頁與地區頁，能大幅擴張長尾關鍵字與 Event rich results。

推薦資料夾結構

/
├── index.html          # 主清單頁 (保持現有)
├── events/
│   ├── [slugified-event-name].html   # 每場活動獨立頁
├── instructors/
│   ├── [lecturer-slug].html          # 講師頁
├── regions/
│   ├── taipei.html
│   ├── taichung.html
│   └── ... 
├── past.html           # 過期活動
└── sitemap.xml         # 更新涵蓋所有頁

URL 規劃：/events/活動標題-slug、/instructors/講師名 等，乾淨且語意化。

實作方式：擴充 build_site_json.py，使用 Jinja2 或 string template 為每筆 event 生成 HTML 檔案。Actions 每日 build 時一併產生。

內部連結：主頁卡片連結到獨立頁；sitemap 更新；JSON-LD 改用個別 Event schema。

風險：建置時間略增（77 場仍極輕量），但 GitHub Pages 無問題。

3其他高 CP 值建議

  • 自訂網域：若可能，綁定 satir-daily.tw 等（免費 DNS），提升品牌信任。
  • 內容擴充：build 時注入更多薩提爾知識文章或講師 bio。
  • 內部連結與麵包屑：強化站內導航。
  • 反向連結：繼續聯絡機構請求連結。
